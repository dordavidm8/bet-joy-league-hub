# פיצ'רים מפורטים

## 1. מערכת ניחושים (Betting)

### איך זה עובד
1. ESPN API שולף משחקים + סיכויים אמיתיים
2. Backend מייצר 2–3 שאלות לכל משחק (winner, total goals, BTTS)
3. משתמש בוחר תשובה + סכום נקודות
4. לאחר סיום המשחק: cron job מסיים את ההניחושים וחולק נקודות

### שאלות אוטומטיות לכל משחק
- **Winner** – מי ינצח? (3 אפשרויות: בית / תיקו / אורח)
- **Total Goals** – Over/Under 2.5 יעדים
- **BTTS** – שתי הקבוצות יכבישו (כן / לא)

### ענישת ניחוש חי (Live Penalty)
כדי למנוע ניחושים "בטוחים" בזמן חי:

| דקה במשחק | קנס |
|-----------|-----|
| 0–45 | 0% |
| 46–60 | 10% |
| 61–70 | 25% |
| 71–75 | 40% |
| 76+ | נעול (לא ניתן להמר) |

**חישוב תשלום:**
```
potential_win = stake × odds × (1 - live_penalty)
```

### Parlay (שילוב הניחושים)
- בחירת 2+ ניחושים
- הסיכויים מוכפלים
- כל הניחושים חייבים לנצח

### Featured Game (משחק מוצג)
- אדמין מסמן משחק כמוצג + אחוז בונוס
- הסיכויים גדלים בהתאם לבונוס
- מוצג בראש הדף הבית

## 2. ליגות (Leagues)

### 3 פורמטים

#### Pool League
- מנהל קובע דמי כניסה
- מנהל מחלק פרסים ידנית בסוף העונה
- סטנדינגס לפי סה"כ נקודות שנצברו

#### Per-Game League
- דמי השתתפות לכל משחק
- חישוב אוטומטי של סטנדינגס לפי ניצחונות
- ניתן לנתק את עצמך ממשחקים ספציפיים

#### Tournament League
- מקושר לתחרות ספציפית (ליגת אלופים, ליגה לאומית...)
- stake_per_game – הימור קבוע לכל מחזור
- **penalty על אי-ניחוש**: משתמש שלא ניחש בזמן משלם קנס
- prize_distribution – חלוקת פרס: `[{place:1, pct:60}, {place:2, pct:30}, {place:3, pct:10}]`
- `auto_settle` – כשמופעל, הפרס מתחלק אוטומטית עם תום התחרות

### זרימת הצטרפות לליגה
```
1. מנהל יוצר ליגה → invite_code ייחודי
2. שיתוף הקוד / הזמנה ישירה לפי username
3. משתמש מוזמן רואה notification "הוזמנת ל-X"
4. לחיצה על accept → league_member record נוצר
```

### הזמנה ישירה
- מנהל מחפש username
- backend מוצא משתמש → שולח notification מסוג `league_invite`
- המוזמן מאשר/דוחה מתוך מרכז ההתראות

## 3. יועץ AI (Betting Advisor)

### מפרט
- מודל: **Groq LLaMA 3.1 8B**
- שפה: עברית (system prompt בעברית)
- מגבלה: **20 הודעות ליום** למשתמש

### הקשר שנשלח ל-AI
```
- שם המשחק + מועד
- ניקוד נוכחי (אם חי)
- שאלות ניחוש זמינות + סיכויים
- היסטוריית הצ'אט (עד 10 הודעות אחרונות)
```

### System Prompt (עיקרי)
> אתה יועץ הניחוש הידידותי של Kickoff. תעזור למשתמש לנתח משחקים ולקבל החלטות ניחוש. אל תיתן ערבויות, אבל הצע ניתוח מבוסס נתונים.

### rate limiting
```sql
-- advisor_usage table
SELECT message_count FROM advisor_usage
WHERE user_id=$1 AND date=CURRENT_DATE
-- אם >= 20 → return 429
-- אחרת → increment + forward to Groq
```

## 4. מיני-גיימס (Daily Mini-Games)

נוצרים אוטומטית כל לילה ב-00:00 UTC.

### 1. Missing XI
- שולף תרכובת קבוצה ממשחק ESPN (14 ימים – 3 שנים אחורה)
- משתמש ממלא 11 שחקנים חסרים מרשימה

### 2. Who Are Ya
- שחקן ספציפי, תמונתו מטושטשת עם Sharp
- עם כל ניחוש שגוי התמונה מתחדדת קצת
- נתוני שחקן מ-Wikidata

### 3. Guess the Club
- לוגו קבוצה מטושטש
- משתמש מנחש את שם הקבוצה

### 4. Box2Box Grid
- רשת 3×3
- כל תא: כל שחקן שמילא את שני הקריטריונים (קבוצה/תחרות)
- מונע כפילויות בין ימים

### 5. Career Path
- רשימת תחנות הקריירה של שחקן (ללא שם)
- משתמש מנחש מי השחקן

### 6. Trivia
- שאלת טריוויה אחת
- אפשרויות A/B/C/D

### מנגנון ניקוד
- כל חידה: עד 100 נקודות
- מינוס נקודות לכל ניסיון שגוי
- UNIQUE constraint: ניסיון אחד בלבד ליום לכל חידה
